## Core Data Model (`model/rule.go`)

```go
type Protocol string  // "tcp" | "udp" | "icmp" | "all"
type Target   string  // "ACCEPT" | "DROP" | "REJECT" | "LOG"
type ChainName string // "INPUT" | "OUTPUT" | "FORWARD"

type Rule struct {
    Comment        string
    Protocol       Protocol
    SrcIP, DstIP   string   // CIDR or empty
    SrcPort, DstPort string // port number/range or empty (requires tcp/udp)
    InIface, OutIface string
    ConntrackState string   // e.g. "ESTABLISHED,RELATED"; when set adds -m conntrack --ctstate
    Target         Target
}

type Chain struct {
    Name   ChainName
    Policy Target  // ACCEPT or DROP
    Rules  []Rule
}

type Ruleset struct {
    ProfileName string
    Input, Output, Forward Chain
}
```

## Device Profile Defaults (`model/profile.go`)

| Profile | INPUT policy | FORWARD policy | Default INPUT rules |
|---------|-------------|----------------|---------------------|
| Desktop | DROP | DROP | conntrack ESTABLISHED/RELATED, loopback, ICMP |
| Server  | DROP | DROP | conntrack, loopback, ICMP, SSH:22, HTTP:80, HTTPS:443 |
| Router  | DROP | DROP | conntrack, loopback, ICMP, SSH:22 on eth0; FORWARD: conntrack + LAN→WAN |
| Custom  | ACCEPT | ACCEPT | (empty — blank slate) |

All profiles: OUTPUT policy ACCEPT.

## Screen State Machine (`ui/app.go`)

```
screenProfilePicker → screenRuleManager → screenRuleEditor
                                       → screenExportPreview
```

Root `AppModel` fields:
- `screen screenState`
- `ruleset model.Ruleset`
- `activeChain model.ChainName`
- `editTarget *model.Rule` (nil = new rule)
- `editIndex int`
- sub-models: picker, manager, editor, preview

Navigation via typed tea messages (`NavigateTo`, `ProfileChosen`, `RuleSaved`, `RuleDeleted`, `PolicyChanged`). Sub-models never touch global state directly — they return commands.

## iptables-save Export (`render/export.go`)

```
# Generated by iptables-interface on <timestamp>
*filter
:INPUT DROP [0:0]
:OUTPUT ACCEPT [0:0]
:FORWARD DROP [0:0]
-A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
-A INPUT -i lo -j ACCEPT
...
COMMIT
```

Flag build order per rule: protocol → src IP → dst IP → in-iface → out-iface → src-port → dst-port → conntrack → comment → target.

`WriteToFile(rs Ruleset, path string) error` — writes with mode `0600`.
